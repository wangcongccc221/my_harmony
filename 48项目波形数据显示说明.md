# 48项目波形数据显示说明

## 一、48项目中的波形数据

### 1. 数据来源
✅ **是的，48项目中的波形显示是下位机发送的真实数据**

- **命令ID**: `FSM_CMD_WAVEINFO` (0x1004)
- **数据结构**: `StWaveInfo`
- **数据流向**: FSM/WAM → HC (下位机 → 上位机)

### 2. StWaveInfo 结构体定义

```cpp
struct StWaveInfo
{
    int nChannelId;              // 通道ID
    ushort waveform0[256];       // 波形数据0（256个采样点）
    ushort waveform1[256];       // 波形数据1（256个采样点）
    float fruitweight;            // 水果重量
};
```

**数据大小**：
- `nChannelId`: 4字节 (int)
- `waveform0[256]`: 512字节 (256 × 2字节)
- `waveform1[256]`: 512字节 (256 × 2字节)
- `fruitweight`: 4字节 (float)
- **总计**: 1032字节

### 3. 数据处理流程

```
TCP服务器接收数据
    ↓
FSM_CMD_WAVEINFO (0x1004) 命令
    ↓
GlobalDataInterface::TcpSvrStatSetBuffer()
    ↓
解析 StWaveInfo 结构体
    ↓
添加到 globalIn_wavelist（最多保存100组）
    ↓
DataProcessThread 处理
    ↓
发出 UpWaveInfoEvent 事件
    ↓
WaveCaptureForm::OnUpWaveInfo() 接收
    ↓
绘制波形图（使用 waveform0 和 waveform1 数组）
```

### 4. 48项目中的波形显示位置

**`WaveCaptureForm`（波形捕捉窗体）**：
- 位置：工程设置 → 重量设置 → 波形捕捉
- 功能：显示真实的重量传感器波形数据
- 数据来源：`GlobalDataInterface::globalIn_wavelist`
- 显示内容：
  - `waveform0[256]` - 绘制为绿色波形线（上半部分）
  - `waveform1[256]` - 绘制为绿色波形线（下半部分）
  - `fruitweight` - 显示在 `FruitWeightTextBox1/2/3` 中（最近3个水果的重量）

### 5. 波形数据特点

- **采样点数**: 每个波形256个点
- **数据范围**: 
  - 12位精度：0-4096
  - 16位精度：0-65535
- **存储数量**: 最多保存100组波形数据
- **更新方式**: 实时接收，每次接收一组新的波形数据

## 二、HarmonyOS项目中的液体卡片波浪

### 1. 当前实现

❌ **不是真实数据，是UI动画效果**

- **组件**: `WaveCard`（波浪卡片组件）
- **数据来源**: 本地动画生成（使用 `WavePhaseManager` 管理相位）
- **显示条件**: 当液体卡片没有数据时，显示装饰性波浪动画

### 2. 液体卡片的数据来源

**真实数据来自 `FSM_CMD_STATISTICS`**：
- `nExitCount[]` - 各出口的水果个数
- `nExitWeightCount[]` - 各出口的重量
- `ExitBoxNum[]` - 各出口的箱数

**波浪图只是装饰**：
- 当卡片有数据时：显示数据列表
- 当卡片无数据时：显示波浪动画（装饰效果）

### 3. 代码位置

**液体卡片区域**: `LiquidCardsArea.ets`
```typescript
enableWave: this.isWaveAnimationEnabled() && 
            idx < this.getWaveCount() && 
            (!cardData.thirdLayer.chartData || cardData.thirdLayer.chartData.length === 0)
```

**波浪卡片组件**: `WaveCard.ets`
- 使用 `WavePhaseManager` 管理动画相位
- 使用正弦函数生成波浪形状
- 纯UI动画，不依赖真实数据

## 四、48项目波形数据详细说明

### 1. 命令处理

**`GlobalDataInterface::TcpSvrStatSetBuffer()`**:
```cpp
case FSM_HC_COMMAND_TYPE::FSM_CMD_WAVEINFO:
{
    StWaveInfo waveinfo = StWaveInfo();
    waveinfo = *((StWaveInfo*)data.data());
    QMutexLocker locker(m_wavelistMutex);
    if (globalIn_wavelist.count() >= 100) // 保存数据不超过100组
    {
        globalIn_wavelist.removeAt(0);
    }
    globalIn_wavelist.append(waveinfo);
    break;
}
```

### 2. 事件触发

**`DataProcessThread::run()`**:
```cpp
case FSM_CMD_WAVEINFO:
    emit UpWaveInfoEvent();
    break;
```

### 3. 波形绘制

**`WaveCaptureForm::OnUpWaveInfo()`**:
- 从 `GlobalDataInterface::globalIn_wavelist` 获取波形数据
- 使用 `waveform0[256]` 和 `waveform1[256]` 绘制两条波形线
- 显示最近3个水果的重量（`fruitweight`）

### 4. 波形捕捉控制

**开启波形捕捉**:
- 发送命令：`HC_WAM_CMD_WAVE_FORM_ON` (HC → WAM)
- 下位机开始发送波形数据

**关闭波形捕捉**:
- 发送命令：`HC_WAM_CMD_WAVE_FORM_OFF` (HC → WAM)
- 下位机停止发送波形数据

## 五、HarmonyOS项目如果要显示真实波形

如果要让HarmonyOS项目的液体卡片显示真实波形数据（类似48项目），需要：

1. **实现 `FSM_CMD_WAVEINFO` 命令处理**：
   - 创建 `StWaveInfo` 接口定义
   - 创建 `TcpWaveInfoParser` 解析器
   - 创建 `WaveInfoDataHandler` 处理器

2. **修改 `WaveCard` 组件**：
   - 接收真实的 `waveform0[256]` 和 `waveform1[256]` 数据
   - 根据真实数据绘制波形，而不是使用正弦函数动画

3. **数据存储**：
   - 使用 `AppStorage` 或管理器存储波形数据列表
   - 类似48项目的 `globalIn_wavelist`（最多100组）

4. **UI更新**：
   - 当接收到新的波形数据时，更新对应的液体卡片
   - 根据通道ID匹配到对应的卡片

---

## 六、总结

- **48项目**: 波形显示是**真实数据**，来自下位机的 `FSM_CMD_WAVEINFO` 命令
- **HarmonyOS项目**: 液体卡片的波浪是**UI动画装饰**，不是真实数据
- **液体卡片的数据**: 来自 `FSM_CMD_STATISTICS` 的出口统计数据（个数、重量、箱数）
